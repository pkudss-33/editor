# Editor 技能 — 使用说明

## 这个技能做什么

**Editor** 是一个为周期性专栏写作工作流设计的战略型会前编辑审阅系统。它将一篇草稿和会议背景转化为结构化的三部分编辑诊断，让两位写作伙伴在周会前读到同一份审阅——讨论从共同背景出发，而非从原始的第一反应开始。

每次审阅输出包含三大支柱：

| 部分 | 方法 | 目的 |
|------|------|------|
| **战略文章审阅** | 麦肯锡式 7 问 | 判断文章是否在战略上清楚有力，而非仅仅写得通顺 |
| **虚拟专家圆桌** | `ljg-roundtable` 方法，3–5 位专家视角 | 从多个智识框架挑战假设，深化论证 |
| **编辑检查清单** | 成熟编辑实践（15 项标准） | 在发表前发现阻碍性问题、风险和高价值改进点 |

这个技能对"什么是好的编辑"有自己的判断：它把核心主张、读者价值和可发表性放在文字润色之上。它区分"发表前必须改"、"改了会更好"和"未来选题思路"。它尽量保留作者的声音。

---

## 什么时候用

当你处于或准备进入**专栏编辑工作流**时，随时调用这个技能。技能描述会在你提到以下关键词时自动触发：

- 编辑 / 改稿 / 编辑审阅
- 草稿反馈 / 文章审阅 / 会前预读
- 每周专栏会议 / 每周复盘 / 共同写作
- 经济评论或观点文章评估
- 专家视角 / 虚拟专家圆桌 / 圆桌审阅
- 麦肯锡式审阅 / 战略文章诊断
- 专栏规划 / 下一篇选题

**具体场景：**

- 你有一篇草稿，周会马上要开 → 生成一份会前预读，让双方在会前对齐
- 你不确定一篇文章是否达到发表标准 → 获取结构化的可发表性诊断
- 你想从多个专家角度压力测试论点 → 获取挑战假设的虚拟圆桌讨论
- 你正在规划下一篇专栏选题 → 获取结构化的选题评估和推荐
- 你收到了一篇模糊或情绪化的草稿初评 → 用冷静的战略诊断替代

## 怎么调用

### 直接调用

```
/editor
```

然后根据提示提供文章草稿和会议背景。技能会自动走完三部分审阅流程。

### 自然语言触发

任何匹配技能触发词的表述都会自动加载：

> "帮我审阅一下这周的专栏草稿"
>
> "Run an editorial review on this draft before tomorrow's meeting"
>
> "用专家圆桌的方法压测这篇文章的论点"
>
> "这篇文章的结构诊断和证据层级校准"

### 附带文章正文

直接把文章内容连同请求一起贴进来：

> "这是本周的草稿——帮我做一份会前编辑审阅。目标读者是财新专栏读者，明天上午 10 点开会，需要决定是发表还是重构。"

## 需要提供哪些信息

技能需要三项信息才能产出有用的审阅。如果缺少任何一项，它会主动询问。

| 输入 | 示例 |
|------|------|
| **文章正文或草稿** | 全文，或文件路径/链接 |
| **目标读者和发表场景** | "财新观点专栏"、"FT 中文网"、"内部战略备忘录" |
| **会议需要做出的决定** | 发表 / 修改 / 重构 / 拆分 / 暂缓 / 继续发展 |

可选但会显著提升审阅质量：

- **作者名字或声音特点** — 帮助技能保留作者声音，而非统一化为通用语调
- **已知的编辑顾虑** — "我们担心证据部分太薄"或"主编质疑了时效性"
- **之前的审阅或反馈** — 让技能追踪哪些问题已经被解决
- **期望的审阅深度** — 快速战略检查 vs. 完整三部分审阅含详细圆桌附件

## 你会得到什么

### 主审阅（始终产出）

一份结构化的会前文件，包含：

1. **页头信息块** — 论点、读者收益、建议的会议决定、一句话编辑判断
2. **7 问战略诊断** — 答案先行、So what、MECE 结构、议题树、证据层级、问题解决路径、会议战略综合
3. **专家圆桌结论** — 最锋利的挑战、最佳深化机会、缺失视角、专家会要求的证据、会议问题
4. **编辑检查清单** — 必须修改、可以加强、风险提示
5. **会议问题** — 帮助作者决定方向的问题，而非触发即时反应的问题
6. **建议议程** — 三项会议结构

### 圆桌附件（默认：回复中内联附录；文件模式：单独的 `.md` 文件）

完整的虚拟专家圆桌，包含 3–5 位专家视角的分轮辩论：

- 第一轮：定义与第一性原理
- 第二轮：核心分歧，含 ASCII 结构图
- 第三轮：对文章的启发
- 知识网络图
- 给作者的开放问题

## 使用示例

### 示例 1：标准周会审阅

**用户：**
> /editor
> 这是本周的专栏草稿，目标读者是财新专栏读者，明天上午 10 点开会，需要决定是发表还是需要大改。
> [贴上文章正文]

**技能响应：**
1. 确认审阅框架（文章、读者、决定）
2. 运行三部分审阅
3. 产出主会前审阅文件
4. 附上虚拟专家圆桌作为附录

**预计时间：** 审阅正文在一次回复中生成；如果圆桌附件非常详细，可能在第二轮中继续输出。

### 示例 2：快速战略检查

**用户：**
> "只做 7 问战略审阅，跳过完整圆桌。时间紧。"

**技能响应：**
仅产出第一部分（战略文章审阅）和第三部分（编辑检查清单），附带简短的专家结论，但不生成详细的圆桌附件。

### 示例 3：选题规划

**用户：**
> "我们需要定下周的选题。有三个候选：[选题 A]、[选题 B]、[选题 C]。"

**技能响应：**
使用**选题规划模板**，从角度、时效性、所需证据、专家视角和风险等维度评估每个候选，然后推荐最强选项，给出暂定标题、核心主张、结构和所需素材。

### 示例 4：仅运行专家圆桌

**用户：**
> "我对文章结构有信心，但想压测核心论点。只跑虚拟专家圆桌。"

**技能响应：**
聚焦第二部分，为文章主题选择 3–5 位专家视角，运行圆桌讨论，产出主审阅结论和详细圆桌附件。

### 示例 5：文件化工作流

**用户：**
> "草稿在 `articles/2026-06-25-trade-war.md`。做完整审阅，圆桌附件单独存为文件。"

**技能响应：**
从文件读取草稿，产出主审阅，并在同一目录下创建 `virtual-expert-panel-trade-war.md` 作为独立附件。

## 常用场景与提示语

### 周会前的标准审阅

> "我们明天下午开周会，讨论这篇稿子 [贴上草稿]。帮我做一份会前编辑审阅，包含完整的战略审阅和专家圆桌。这篇文章的目标是财新专栏，读者是关注宏观经济的专业人士。"

### 结构感觉不对劲时

> "这篇稿子读起来很重复，但我找不出问题在哪。帮我做 MECE 结构诊断和议题树分析。不用管措辞——只告诉我论证支柱是否成立。"

### 论证力度不够时

> "我觉得这篇论证不够强，但说不清哪里弱。请用证据层级（evidence hierarchy）帮我校准，并让专家圆桌告诉我还需要什么数据或分析才能让论证立得住。"

### 准备重写时

> "主编觉得这篇缺乏原创性。帮我做一轮完整的编辑审阅，重点放在：读者收益（reader payoff）、原创性和缺失视角上。专家圆桌至少选一位非经济领域的意外视角。"

### 规划选题时

> "接下来三周的选题还没定。我有四个可能的题目：[列出题目]。帮我用选题规划模板做评估，推荐下周写哪篇。"

### 合作作者之间有分歧时

> "我和合作作者在这篇文章的论点上有分歧——我认为应该更强调风险，她觉得应该强调机会。帮我审阅草稿，把分歧定义为会议需要解决的编辑取舍，不要站队。"

## 技巧与最佳实践

### 1. 提供完整草稿，而非摘要
技能需要诊断结构、证据和论证——它需要看到实际文字才能做到这一点。摘要只能让它给出泛泛的反馈。

### 2. 明确发表场景
"经济媒体专栏"太宽泛了。"财新观点专栏，读者是专业的宏观经济受众"会产生精准得多的反馈。发表场景决定了证据标准、原创性要求和风险容忍度。

### 3. 明确会议需要做什么决定
"这篇可以发表了吗？"和"我们应该重构还是拆成两篇？"会产生不同类型的审阅。这个决定决定了技能优先关注什么。

### 4. 赶时间就跳过圆桌
如果你只有 5 分钟就要开会，只要 7 问战略审阅和编辑检查清单就够了。圆桌可以之后补。

### 5. 把专家圆桌当作讨论起点，而非终审判决
圆桌的设计目的是暴露张力、深化会议讨论。"最锋利的挑战"是需要讨论的，不是需要不加批判地接受的。

### 6. 让技能主动提问
如果你给了草稿但没有背景，它会主动问：目标读者、发表场景、会议目标、期望深度。回答这些问题会让审阅质量大幅提升。

### 7. 在文章之间使用选题规划模板
这个技能不只是用来审阅已完成草稿的。在写之前用**选题规划模板**评估候选题目，可以节省时间、避免薄弱开局。

### 8. 保存圆桌附件以便复用
虚拟专家圆桌附件中包含的框架、张力和开放问题，在单次会议之外也有价值。把它们和文章文件一起保存——它们会成为未来同类选题可复用的智识库存。

## 与其他技能的关系

| 技能 | 关系 |
|------|------|
| **ljg-roundtable** | editor 技能的第二部分（虚拟专家圆桌）采用了 `ljg-roundtable` 的方法来组织专家辩论。圆桌技能提供通用的辩论框架；editor 技能将其专门应用于文章审阅。 |
| **huangwei-report-builder** | 与 editor 技能互补：用 report builder 将零散的研究素材整合为精致的草稿，然后用 editor 技能在发表前审阅该草稿。 |
| **management-report-builder** | 类似的互补关系——先构建报告，再用 editor 技能审阅。 |

---

## 输出语言

技能使用与草稿和用户请求相同的语言产出审阅。中文草稿得到中文审阅；英文草稿得到英文审阅。专家圆桌中的框架和术语（如"MECE"、"nut graf"、"evidence hierarchy"）无论审阅语言如何，均保留原文。
